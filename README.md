<small>
2026 edit: converted to static site for github pages. in retrospect i only used react cuz it's trendy but a website with five pages really does not need to be so OP like that. also it's fun to use platforms like glitch for the social aspect, but at a certain point i wanted to make edits using my own ide, and the glitch setup actually complicated local development supremely, they really wanted you to stay in their web editor... which does make sense... all this immaterial media is so ephemeral anyway.
</small>

<small>
oh, and a meta-narrative: the readme file as the site for a ramble better suited to social media. i actually like the slow aspect of this, the lack of immediate feedback. i will never know who's read this and that is somehow deeply satisfying. i stopped posting rambles like this to socials pretty quickly because something about the navel gazey way it kept me checking my phone did not sit well with me. and like, getting sucked into a vortex of short form recommendations really numbs your brain. pretty much every mainstream platform has this immediate feedback problem now, and i'm tempted to indulge the urge to become an internet hermit, but then i remember how easy it was to make youtube have no ads, or make tumblr show posts chronologically, and i wonder if we can all just jailbreak our phones and fix these problems ourselves. it really shouldn't be that hard? i say, with no context. i would literally buy a different phone in order to do that. an idea for another day.
</small>
